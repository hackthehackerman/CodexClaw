# CodexClaw v0

## Goal

Ship a thin, installable messaging bot on top of `codex app-server` that a user can:

- install with `npm`
- configure for Telegram or iMessage
- run locally with safe defaults
- use in one or more explicitly allowed chats

## Product Experience

The v0 experience should be:

1. `npm install -g codexclaw`
2. `codexclaw init`
3. user follows a short onboarding flow for Telegram or iMessage
4. `codexclaw start`
5. the bot is usable immediately in the configured chat

## Must Have

- installable npm package
- local config generated by `codexclaw init`
- clear setup path for Telegram Bot API
- clear setup path for iMessage via BlueBubbles
- safe default policy:
  - `policy.default = "deny"`
  - narrow allow rules
  - `allow_self_messages = false`
  - `approval_policy = "untrusted"` unless explicitly changed
  - `sandbox = "workspace-write"` unless explicitly changed
- one conversation maps to one Codex thread
- per-conversation turn serialization
- local control page for basic status and debugging
- support for text plus basic media flows already in the harness

## Remaining Before Release

- make `codexclaw init` feel like true onboarding instead of only template generation
- verify one real Telegram setup from a fresh config
- verify one real iMessage setup from a fresh config
- do a final release pass on docs, npm package contents, and public repo hygiene
- publish to npm and push the release commit

## Out of Scope

- WhatsApp
- richer presence / typing indicators
- broader policy presets beyond a narrow safe setup
- first-class tasks and automations
- advanced operator UI beyond the current control page
- plugin ecosystem or generalized multi-agent orchestration
- polished admin command surface

## Standard for v0

If a new user can install CodexClaw, configure one transport, start it, and successfully message the bot without touching source code, v0 is good enough to ship.
